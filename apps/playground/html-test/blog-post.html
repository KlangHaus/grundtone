Hvad er forskellen på palette- og semantiske tokens?

Palette-tokens er rå farveværdier (blue-600, gray-100). Semantiske tokens beskriver funktion (color-primary, color-surface). Komponenter bør kun bruge semantiske tokens — det gør theming og dark mode trivielt.

ℹ️

Semantiske tokens som --color-primary og --color-surface tilpasser sig automatisk til dark mode via CSS light-dark().

Alle tokens eksponeres som CSS custom properties på :root: 

:root {
  --color-primary: light-dark(#0059b3, #4dabf7);
  --color-surface: light-dark(#f8f9fa, #1e1e1e);
  --shadow-md: 0 4px 6px -1px rgba(0, 0, 0, 0.1),
               0 2px 4px -1px rgba(0, 0, 0, 0.06);
}
⚠️

Undgå at bruge palette-farver direkte i komponenter. De er beregnet til at bygge temaer — ikke til at style UI.

Ofte stillede spørgsmål

Brug CSS custom properties i dine komponenter — de er runtime-dynamiske og understøtter theming. SCSS-variabler er build-time og kan ikke ændres af brugeren.

Sæt data-theme="dark" på <html>. Alle semantiske tokens bruger light-dark() CSS-funktionen, så farverne skifter automatisk.

Ja — overskriv eksisterende tokens via CSS custom properties på :root, eller tilføj nye via SCSS med @grundtone/design-system/scss/lib.
